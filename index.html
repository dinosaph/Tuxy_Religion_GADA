1. Abstract

The following article presents you the development of a service-based web game in which users are presented two main types of games, a claw catcher machine game and a foosball one. The application was developed during InfoIasi College Web Application Development Course, in autumn-winter 2022 and it uses html, css, js (frontend) and game characters related APIs and ontology technologies.

2. Introduction

The project's starting idea was that we need a service-based web game that uses data about characters provided by several APIs (like Marvel Comics API and Pok√©API), various player interactions, game rules, characteristics and assets modeled and accessed by RDF.

GADA by Tuxy Religion gladly presents an idea that combines two types of games that you can't stop playing: a claw catcher machine game and a foosball machine game. The user receives initial tickets that he can use in the claw catcher game in order to unlock new hero characters. These characters are further used in the foosball matching game (user vs computer), game in which the user wins more tickets.

Our game promises never-stopping & colorful fun!

3. Architecture

GADA is proposed to be developed in the following 3 modules:

  • User Interface Web & Game - Minimalist and friendly design
  • Server - Main backend component that deals with all actions between the modules
  • Game - Fun, colorful game combining 2 classic machine games: claw catcher and foosball

    • 3.1 UI Web & Game

    The users can navigate on 3 pages:

  • Page - Play - where they can play our game
  • Page - About - where they can read details of the project, team and gameplay
  • Page - Contact - where they can keep in touch with the development team

    • Used technologies: html, css, js, canvas

    GADA's server purpose is to provide website/web pages to users with necessary game and characters data. Also, it communicates with APIs for requested characters data.

    4. Data Modelling
